Accountability Framework

A Three-Tier Verification Chain

No single person can submit and approve their own work. Every checklist passes through three independent checkpoints before it is considered fully verified.

🧹
Employee
Selects name from roster, completes shift checklist for their area, submits with compliance score
Submits Record
👔
Lead
Reviews all submissions for their building that day. Signs off with initials. Flags any gaps
Daily Sign-Off
🖊
Supervisor
Reviews full week across all buildings. Comments on every gap. Signs with drawn signature
Weekly Sign-Off
📋
Auditor
Gets PDF sign-off, CSV audit export, and ISO Excel sheet — one file per request, fully attributed
Audit-Ready
Why this matters for ISO compliance: The chain creates a documented accountability trail — Record ID → Employee → Lead sign-off → Supervisor sign-off with gap acknowledgements → Export with worker initials in cells. An ISO auditor can trace any exported cell back to the exact person, shift, and date.

Access Control

Five Role-Based Access Levels

Each role has its own entry point, dedicated views, and scoped capabilities. No role can exceed its permissions — even with direct URL access.

🧹
Shop Floor
Employee
Shift workers complete daily sanitation checklists for their assigned area. Name-picker login — optimised for shared tablets on the factory floor.
  • Select name from active roster (no password needed)
  • Navigate production areas by building
  • Complete D / A / N shift checklists
  • See supervisor priority notes for their area
  • Submit with automatic compliance scoring
  • Works fully offline — syncs when reconnected
👔
Lead
Lead Verifier
Building leads perform the daily verification layer — reviewing all employee submissions for their building before the supervisor's weekly sign-off.
  • View all submissions for their building by day
  • Per-record detail: compliance %, all item responses
  • Verify each record — pass/fail per item, optional note
  • Draw signature for daily building sign-off
  • Flag gaps for supervisor awareness
  • Cannot see other buildings or admin functions
🖊
Supervisor
Supervisor
Plant supervisors own the weekly sign-off cycle — reviewing all buildings, acknowledging every gap in writing, and signing with a drawn signature.
  • Weekly sign-off with drawn signature pad

  • Area-specific notes with due dates (Priority Tasks)
  • PDF sign-off export (jsPDF — filename encoded with week + shift)

  • Audit export history log (filename, filters, timestamp)

Security Analysis

Zero-Vulnerability Build

  • ✓
    ExcelJS replaces SheetJS / xlsx 0.18.5 CVE Removed The old xlsx library (SheetJS 0.18.5) had two active CVEs: Prototype Pollution (High severity) allowing arbitrary code execution, and ReDoS (High) causing denial of service via malformed strings. ExcelJS has zero known CVEs and active security maintenance.
  • ✓
    @tanstack/react-start — all undici CVEs patched 4 Fixed Resolved four transitive vulnerabilities in undici (HTTP client): TLS certificate bypass (High), WebSocket DoS (High), cross-origin SOCKS5 proxy exposure (Medium), and HTTP header injection (Medium). All patched in the current build.
  • ✓
    TypeScript end-to-end — strict type safety All inputs, API responses, and data structures are typed. No untyped any in core logic paths. TypeScript catches data-shape mismatches at build time, not runtime.
  • ✓
    Validated input at every boundary — Vitest unit tests Roster library rejects names over 80 characters and any containing < or > to prevent injection. getInitials() is tested with 10 edge cases covering null, undefined, empty string, non-string, whitespace, and multi-word inputs — all verified by Vitest.
  • ✓
    No third-party tracking, ads, or external data calls The application makes zero external data calls at runtime. No analytics SDK, no ad network, no telemetry. All data stays on the device. The JavaScript bundle contains no API keys or credentials.
  • ✓
    XSS-safe by construction — React JSX rendering React's JSX escapes all user content before inserting into the DOM. No dangerouslySetInnerHTML in the codebase. No eval(), no inline event handlers.
  • ⚠
    Pilot-stage: localStorage data storage (no server encryption) Current pilot uses localStorage — appropriate for single-device demonstration. Phase 2 adds Supabase (AES-256 at rest, TLS 1.3 in transit, SOC 2 Type II). No data leaves the device until backend is connected.

Audit-Ready Output

Three Export Formats — All Audit-Grade

Every export is named, filtered, timestamped, and logged. An auditor can request any record in any format — instantly.

📗
ISO Excel Export
ExcelJS · .xlsx · Zero CVEs

Exact Ferrero ISO sheet format — multi-tab workbook, dark navy headers, worker initials in cells. Matches paper sheets but fully digital and attributed.

  • One tab per building/area
  • Header block: Area, Code, Revision, Approved By
  • Worker initials in each cell (e.g. "JS" = John Smith)
  • Green fill = completed, orange = if-needed
  • Pre-download preview modal before generating
  • Week picker, comments toggle, completed-only toggle
  • Gold spinner + locked buttons during generation
📄
Supervisor PDF Sign-Off
jsPDF · .pdf · Signed

Weekly sign-off document including supervisor name, drawn signature, week dates, all gap acknowledgements with comments — ready for filing.

  • Supervisor name + drawn signature embedded
  • Week date range (Mon–Sun) in header
  • Building filter and shift filter applied
  • Every gap acknowledgement comment included
  • Filename: weekly-signoff_[week]_b-[bldg]_s-[shift]_[ts].pdf
  • Export attempt logged to audit history
📑
CSV Audit Export
Native CSV · .csv · Filterable

Full audit-grade CSV of all submitted records for any date range, filterable by building and shift. Ready for Excel, Tableau, or filing system import.

  • Columns: Record ID, employee, area, building, shift
  • Date, time taken, compliance %, all item responses
  • Filter by building and shift before export
  • Timestamped filename with filter params encoded
  • Exports queued if one is already in progress
  • Full export history log (who exported, when, what)
Export History Log

Every export attempt is recorded: filename, filters used, timestamp, success/failure status. An audit trail of who exported what and when.

Queued Processing

Exports requested while a previous one is in progress are queued — not dropped. No data loss, no duplicate files.

Settings Persisted

Week range, building filter, and toggle preferences are saved in localStorage. Your export configuration is remembered between sessions.

Offline Capability

Works Without Internet — No Submissions Lost

Ferrero is built for the factory floor where connectivity is unpredictable. Checklists submitted offline are queued and automatically synced when the network returns.

  • 📡
    Built-in Offline Sync Worker Background worker runs on app startup and listens for the browser's 'online' event. Drains the queue automatically when connectivity returns.
  • 🔄
    Cross-Session Persistence Records queued in a previous browser session are not lost. They are re-queued on next app open and synced immediately.
  • 🟢
    Toast Confirmation on Sync When pending records drain successfully, a toast notification confirms exactly how many records were synced — with gold spinner during processing.
  • 🏭
    Factory Floor Ready Operations are never blocked by a network outage. Every employee can complete and submit their checklist regardless of connectivity.
